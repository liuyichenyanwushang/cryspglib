## Project Overview

This is a Rust implementation of the spglib (space group library) for crystallographic symmetry analysis. The library provides functionality for space group identification, symmetry operations, primitive cell reduction, k-point generation, and related crystallographic computations. The project includes a complete C reference implementation in `src_c/` and Python data conversion scripts.

## Common Development Tasks

### Building the Rust Library
```bash
# Build the library in debug mode
cargo build

# Build in release mode
cargo build --release

# Run all unit tests (defined in src/lib.rs)
cargo test

# Run tests with verbose output
cargo test -- --nocapture
```

### Building the C Library (Reference Implementation)
```bash
cd src_c
mkdir build && cd build
cmake .. -DSPGLIB_INSTALL=OFF  # Adjust CMake options as needed
make
```

### Running Integration Tests
The repository contains standalone test binaries in the root directory (e.g., `test_debug.rs`, `test_delaunay.rs`). These can be compiled and run manually:

```bash
# First, build the Rust library
cargo build

# Compile and run a test (example for test_debug.rs)
rustc --extern rspglib=target/debug/librspglib.rlib test_debug.rs -o test_debug
./test_debug
```

### Data Conversion


## High-Level Architecture

### Module Dependencies
The library follows a layered architecture with the following key modules:

```
mathfunc (linear algebra)
    ↓
cell (crystal cell representation)
    ↓
symmetry (symmetry operations) ← spg_database (space group data)
    ↓
spacegroup (space group analysis)
    ↓
primitive (primitive cell reduction)
    ↓
pointgroup (point group analysis)
    ↓
kgrid/kpoint (Brillouin zone sampling)
```

### C Reference Implementation

The Rust implementation closely mirrors the C reference implementation in `src_c/`. Function names, algorithms, and data structures are adapted from the C version. The C code serves as both a reference and source for data conversion via Python scripts.

### Key Architectural Patterns

1. **Data Flow**: Most operations start with a `Cell` object, which is processed through symmetry analysis to produce space group information, then reduced to primitive cells, and finally used for k-point generation.

2. **Database-Driven Design**: Space group data is stored in `spg_database.rs` (733KB auto-generated file). Hall symbol data is in `hall_symbol.rs`. These databases are converted from C source files using Python scripts.

3. **Numerical Precision**: The library uses double-precision floating point (`f64`) for lattice vectors and positions, with integer matrices (`Mat3I`) for symmetry operations. Constants like `INT_PREC` (0.1) and `ZERO_PREC` (1e-10) control numerical tolerance.

4. **Reduction Algorithms**: Multiple reduction algorithms are implemented:
   - Niggli reduction (`niggli.rs`)
   - Delaunay reduction (`delaunay.rs`)
   - Primitive cell reduction (`primitive.rs`)

5. **Symmetry Search**: The main entry point is `spa_search_spacegroup` in `spacegroup.rs`, which orchestrates symmetry detection, space group matching, and cell reduction.

### Important Data Structures

- `Cell`: Container for lattice vectors, atomic positions, and atom types
- `Symmetry`: Collection of symmetry operations (rotations + translations)
- `SpacegroupType`: Space group information including number, Hall symbol, and centering type
- `Primitive`: Reduced primitive cell with transformation matrix
- `Kgrid`: k-point mesh in reciprocal space

### File Organization

- `src/`: Rust source code (main implementation)
- `src_c/`: C reference implementation (spglib)
- `txtfile/`: Copies of Rust source files (generated by `do.sh`)
- Root directory: Test binaries and conversion scripts

### Testing Strategy

- **Unit tests**: Standard Rust `#[test]` functions in source files
- **Integration tests**: Standalone binaries in root directory
- **Reference tests**: Comparison with C implementation outputs
- **Database tests**: Verification of converted data integrity

## Development Notes

- The `spg_database.rs` file is auto-generated; manual edits will be overwritten by conversion scripts.
- The C implementation in `src_c/` serves as both reference and source for data conversion.
- Numerical tolerances are critical for symmetry detection; adjust `INT_PREC` and related constants carefully.
- When adding new space group data, update the C source files and regenerate Rust files using Python scripts.
